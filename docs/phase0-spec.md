## 1. この組み合わせは成立するか？

### 結論: 成立する。ただし「ハイブリッドモデル」で。

| 組み合わせ | 判定 | 理由 |
|---|---|---|
| ノベルゲー × 教育 | ◎ | 批判的思考スキル向上の学術エビデンスあり（Ecuador研究: 16%→46%） |
| マダミス × ノベルゲー | △ | マダミスの本質（秘密保持・4人協力・演技）はソロVNで再現不可 |
| **教育マダミス × ノベルゲー** | **◎** | 「VNで感情を動かす → 教室で議論する」の2段階設計なら両方の強みが活きる |

### なぜ「ハイブリッド」か

```
【現在のマダミス】
  4人がカードを読む → 議論する → 投票する
  問題: テキストだけで没入感が弱い

【フルノベルゲー化】
  1人でVNを読む → 選択肢を選ぶ → エンディング
  問題: 協働・議論・演技が消える（マダミスの強み8割消滅）

【ハイブリッドモデル（採用）】
  VNで物語を体験する → 問いかけで立ち止まる → 自分の考えを選ぶ/書く
  教室では: 先生がVNを投影 → 生徒がChromebookで回答 → クラス全体で議論
  自宅では: 1人でVNを読む → 選択肢で自分の考えを試す → 振り返りを読む
```

### 採用する要素 / 捨てる要素

| 採用（VNから） | 理由 |
|---|---|
| タイプライター演出テキスト | 情報の重要度を演出で伝えられる |
| キャラ立ち絵 + 表情変化 | 感情移入が格段に上がる |
| 背景画像 | 「学校の廊下にいる」臨場感 |
| BGM（場面別） | 日常→緊張→感動の感情弧を設計できる |
| SE（効果音） | 「ガシャーン！」等で没入感 |
| 選択肢UI | 「自分で考える」瞬間を作れる |
| フェーズ転換演出 | 証拠公開・クライマックスの緊張感 |

| 捨てる（VNから） | 理由 |
|---|---|
| 分岐シナリオ / マルチエンド | 教室で全員が違うルートだと授業崩壊。制作コスト非現実的 |
| ノンストップ議論（ダンロン） | アクション要素。Chromebookブラウザでは過大な開発コスト |
| フルボイス | 30人が各自端末で聞くと教室が騒音地獄 |
| 処刑演出 | 小学校教育上、不適切 |
| リプレイ用マルチルート | 教育マダミスは「1回の体験」が本質 |

## 2. ノベルゲーム用語辞典（人間シミズ向け）

このプロジェクトで使う用語を、マダミス制作者の言葉で説明する。

| VN用語 | マダミスで言うと | 具体例 |
|---|---|---|
| **シーン (Scene)** | 1枚のカードを読むパート | 「証拠カード①を読む場面」= 1シーン |
| **立ち絵 (Sprite)** | キャラクターの絵。表情差分あり | ハヤトの「普通」「焦り」「泣き」の3枚 |
| **背景 (BG)** | 場所の絵 | 「放課後の廊下」「教室」 |
| **テキストウィンドウ** | 画面下部のセリフ表示欄 | common.mdのテキストが流れる場所 |
| **ネームプレート** | 誰が喋っているかの名前表示 | 「ノゾミ先生」「ハヤト」 |
| **タイプライター効果** | 文字が1文字ずつ現れる演出 | ダンロンの裁判中テキスト表示 |
| **SE (Sound Effect)** | 効果音 | 「ガシャーン」「ドキッ」 |
| **BGM** | 場面ごとの音楽 | 日常曲、緊張曲、感動曲 |
| **選択肢 (Choice)** | プレイヤーが答えを選ぶ画面 | 「一番先に話すべきなのは？」→ 4択 |
| **トランジション** | 画面切り替え効果 | フェードイン、暗転 |
| **スクリプト** | ゲームの台本データ（YAML形式） | common.md → script.yaml に変換 |
| **エンジン** | スクリプトを読んで画面に表示するプログラム | ゲームの骨格部分 |

## 3. moral-01 ゲームフロー

### 全体構成（プレイ時間: 約15-20分）

```
プロローグ ──→ Act 1: 調査 ──→ Act 2: 深掘り ──→ Act 3: 真実 ──→ エピローグ
  (2分)         (5分)          (5分)           (5分)         (2分)

感情曲線:
  穏やか → 衝撃 → 興味 → 共感 → 問いかけ → 感動 → 余韻
```

### プロローグ（2分）

**目的: 「何が起きたか」を感情込みで伝える**

```
[背景: 放課後の廊下。夕日が窓から差し込む]
[BGM: 穏やかな日常曲]

ナレーション:
  「放課後の学校。いつもと変わらない、静かな時間——」

[SE: ガシャーン！（大きな破壊音）]
[画面シェイク（振動演出）]
[BGM: 停止（無音）]

ナレーション:
  「廊下に飾られていた "夢の花びん" が、割れた。」

[背景: 割れた花びんのアップ画像]

ナレーション:
  「去年の6年生が卒業記念に作った、大切なもの。
  　クラスのみんなで土をこねて、色を塗って、窯で焼いた。
  　表面には "夢" という文字と、みんなの手形——」

[立ち絵: ノゾミ先生（心配な表情）登場]
[BGM: 静かな緊張曲]

ノゾミ先生:
  「花びんが割れている。誰か知っている人はいますか？」

[立ち絵: ハヤト・ミルク・マモル・コトハ（4人並び、全員目をそらしている）]

ナレーション:
  「4人とも、だまっている。」

ノゾミ先生:
  「犯人探しをしたいわけじゃないの。
  　でも、このまま誰も何も言わないのは、もっと悲しい。
  　みんなで話し合って、これからどうするかを考えよう」

[画面: タイトル表示「われた花びんとだまった4人」]
```

--- 証拠カード③ 4人の「そのときの事情」 ---

  「4人とも、ウソはついていない。
  　でも——何かを言っていない気がする。」

### Act 2: 深掘り（5分）

**目的: 感情を動かし、問いかけで「考える」瞬間を作る**

```
--- 証拠カード④ 卒業生からの手紙 ---

[BGM: 切り替え → 静かなピアノ曲]
[背景: 花びんの台の裏側]

ナレーション:
  「花びんの台の裏に、手紙が貼ってあった。
  　去年の6年生が残したものだ。」

[画面: 手紙が開かれるアニメーション]
[テキスト速度: ゆっくり（通常の1.5倍の間隔）]

手紙:
  「この花びんは『失敗しても、やり直せる』の証（あかし）です。

  　作っている途中、何度もヒビが入りました。
  　でも、みんなで粘土を足して、もう一度焼きました。

  　完璧じゃないけど、みんなの気持ちが詰まってる。

  　花びんに込めた気持ちは、壊れないよ。」

[SE: 静かな余韻（ハープの一音）]
[画面: 少し間を置く（3秒の無操作時間）]

--- 証拠カード⑤ ノゾミ先生の問いかけ ---

[立ち絵: ノゾミ先生（真剣だが優しい表情）]
[BGM: 静かな緊張曲]

ノゾミ先生:
  「みんなに、考えてほしいことがあるの。」

[画面: 黒板に文字が書かれるアニメーション]

=== 問いかけ① ===
「友だちを助けに行く途中だったら、走ったことは『悪いこと』？」

→ 【選択肢】
  A. 悪くない。友だちを助けるためだから
  B. 悪い。ルールはルールだから
  C. 分からない。どっちの気持ちも分かる

[選択後: 短いナレーションで「こういう考え方もあるよ」と補足]

=== 問いかけ② ===
「本当のことを知っているのに黙っているのは、『やさしさ』？ それとも『ずるさ』？」

→ 【選択肢】
  A. やさしさ。言ったら誰かが傷つくから
  B. ずるさ。本当のことを言うべきだから
  C. どちらでもない。黙る理由による

=== 問いかけ③ ===
「やるべきことがあったのに、別の大事なことをしていた。これは『サボり』？」

→ 【選択肢】
  A. サボりじゃない。もっと大事なことがあった
  B. サボり。当番は当番だから
  C. 場合による。何を優先したかで変わる

=== 問いかけ④ ===
「危険に気づいていたのに『誰かがやるだろう』と放置した。これは『関係ない』って言える？」

→ 【選択肢】
  A. 関係ない。自分の仕事じゃないから
  B. 関係ある。気づいた人には責任がある
  C. 難しい。忙しかったら仕方ない

=== 問いかけ⑤（最重要） ===
「4人の中で、一番先に口を開くべきなのは誰？」

→ 【選択肢】
  A. ハヤト（花びんにぶつかった本人）
  B. ミルク（全部見ていた目撃者）
  C. マモル（見回り当番だった責任者）
  D. コトハ（台の危険に気づいていた人）
  E. 誰が先でもいい。大事なのは全員が話すこと
```

### Act 3: 真実（5分）

**目的: 4人の「本当の事情」を1人ずつ明かし、感情のクライマックスを作る**

```
[BGM: 告白のテーマ（静かだが芯のある曲）]
[画面: 暗転 → テロップ「真実」]

--- ハヤトの真実 ---

[立ち絵: ハヤト（うつむいている → 顔を上げる）]

ハヤト:
  「……オレが走って、花びんにぶつかった。
  　でも——ユウキが転んで血が出てるって聞いて、
  　ばんそうこうを取りに行かなきゃって……。
  　走ったのは悪い。でも、友だちを助けたかったんだ。」

[ナレーション]
  「友だちを助けたい気持ちと、ルール。
  　ハヤトの中で、2つの『正しいこと』がぶつかっていた。」

--- ミルクの真実 ---

[立ち絵: ミルク（怖がっている → 覚悟を決めた表情）]

ミルク:
  「……私、全部見てた。
  　ハヤトくんが花びんにぶつかるところ。
  　目が合ったの。でも、ハヤトくんはそのまま走っていった。」

  「去年、友だちのリナちゃんのことを先生に言ったら、
  　みんなから『チクリ魔』って呼ばれた。
  　……また嫌われるのが怖くて、言えなかった。」

[SE: 静かな衝撃音]
[ナレーション]
  「本当のことを言う『勇気』と、傷つきたくない『防衛』。
  　ミルクは、過去の痛みに縛られていた。」

--- マモルの真実 ---

[立ち絵: マモル（責任感のある顔 → 苦しそうな表情）]

マモル:
  「見回り当番だったのに、持ち場を離れてた。
  　1年生のリコちゃんが泣いてて……教室まで送ったんだ。
  　往復10分。その間に花びんが割れた。」

  「当番を離れたのは事実だから——
  　『お前がいれば防げた』って言われたら、何も言い返せない。」

[ナレーション]
  「目の前で泣いている子を助けることと、当番の責任。
  　マモルは『2つの正しいこと』の間で引き裂かれていた。」

--- コトハの真実 ---

[立ち絵: コトハ（平然 → 目を伏せる）]

コトハ:
  「……先週の金曜日、飼育小屋に行く途中で気づいてた。
  　花びんの台の脚が1本短くて、グラグラしてるって。」

  「でも——『誰かが直すだろう』って思って、
  　そのまま飼育小屋に行った。」

  「……もし私があの時、先生に言ってたら——
  　花びんは割れなかったかもしれない。」

[SE: 沈黙の間（3秒）]
[BGM: 静かに感動的な曲へ移行]

[ナレーション]
  「これを『傍観者効果（バイスタンダー・エフェクト）』という。
  　『誰かがやるだろう』——みんながそう思うと、誰もやらない。

  　コトハは悪い人じゃない。
  　でも、『気づいた人には、特別な責任がある』——
  　それを、この出来事が教えてくれた。」
```

### エピローグ（2分）

**目的: 金継ぎのメタファーで希望を残して終わる**

```
[BGM: 希望のテーマ（明るく優しい曲）]
[背景: 教室。窓から夕日]

[立ち絵: 4人が向かい合っている]

ノゾミ先生:
  「ありがとう。みんなが話してくれて、先生はうれしい。」

  「花びんは割れてしまった。
  　でも、日本には『金継ぎ（きんつぎ）』という技術があるの。」

[画面: 金継ぎの写真 or イラスト]

  「壊れたものを金で繋いで、前よりもっと美しくする。
  　割れた跡が、新しい模様になる。」

  「花びんに込められた気持ちは、壊れない。
  　みんなで正直に話せたこと——
  　それが、この花びんの新しい『金色の線』だよ。」

[画面: エンディングカード]
  「あなたの選択を振り返ろう」
  [プレイヤーが選んだ5つの回答を一覧表示]
  [各選択について、1行の振り返りコメント]

[画面: 終了]
  「ナゾトキ探偵団 答えのない法廷 Vol.1 ——おわり」
```

### VNエンジン コア機能（Phase 0スコープ）

| 機能 | 説明 | 実装方法 |
|---|---|---|
| **SceneManager** | シーンの進行状態を管理する状態マシン | Preact の useState/useReducer |
| **TextRenderer** | タイプライター効果でテキストを表示 | setInterval + CSS animation |
| **SpriteManager** | 立ち絵の表示・切替・フェードイン/アウト | CSS transition + position absolute |
| **AudioManager** | BGM再生・SE再生・フェードイン/アウト | Web Audio API or HTML5 Audio |
| **ChoiceSystem** | 選択肢の表示・選択結果の記録 | Preact コンポーネント + state |
| **TransitionEffect** | 画面暗転・フェードイン等 | CSS animation |
| **ScriptLoader** | YAMLスクリプトを読み込みパースする | js-yaml ライブラリ |

### Phase 0 で作らないもの

| 機能 | 理由 | いつ作るか |
|---|---|---|
| セーブ/ロード | プロトタイプは通しプレイのみ | Phase 1 |
| バックログ（読み返し） | 実装コスト vs 検証価値が合わない | Phase 1 |
| オートモード | プロトタイプは手動送りのみ | Phase 1 |
| スキップ機能 | 初回プレイが前提 | Phase 1 |
| 教室連携（Supabase） | まずソロで検証 | Phase 2 |
| シナリオ変換パイプライン | 手動でYAML作成 | Phase 2 |

### なぜ game/ と site/ を完全に分けるか

| 理由 | 説明 |
|---|---|
| **独立デプロイ** | game/ は静的SPA。site/ はAstro+Vercel。デプロイ先が違う |
| **技術スタックの違い** | game/ は Vite+Preact SPA。site/ は Astro SSR。混ぜると複雑化 |
| **開発サイクルの違い** | game/ は実験フェーズ。site/ は本番運用中。互いに影響させない |
| **人間が迷わない** | 「ゲームのコードは game/」「サイトのコードは site/」。一目瞭然 |

### game/ のデプロイ先（Phase 0）

- **ローカル開発**: `cd game && npm run dev` → localhost:5173
- **Phase 0 検証**: Vercel に別プロジェクトとしてデプロイ（game.nazotoki.gamanavi.com 等）
- **Phase 1 以降**: site/ のサブルート（/game/moral-01）に統合する可能性あり

## 9. Phase 0 の判断基準

### 検証する仮説

> **「VN演出を加えることで、テキストだけのマダミスより没入感・感情的反応が明確に上がるか」**

### 成功基準

| 指標 | 測定方法 | 成功ライン |
|---|---|---|
| **主観的没入感** | テストプレイ後の口頭フィードバック | 「すごい」「もっとやりたい」的な反応 |
| **感情的反応** | Act 3（真実パート）での表情・反応観察 | 明確な感情変化が見られる |
| **教育的到達** | 選択肢の回答分布 | 全員が同じ回答にならない（多様な意見が出る） |
| **技術的実現性** | Chromebook での動作 | 60fps、音声遅延なし |

### 失敗した場合の撤退基準

- 「テキストだけでも十分」という反応 → VN化は演出レイヤー追加に留め、game/ は凍結
- Chromebook で動作が重い → 軽量化 or site/ 内に演出コンポーネント追加に切り替え
- 制作コストが見合わない → BGM + SE だけを site/ に追加する「最小介入」に切り替え
