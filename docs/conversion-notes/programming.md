## programming-01（バグ探偵団 Vol.1）

**変換日**: 2026-03-23

### サマリー
- シーン数: 16 / キャラ数: 5 / 選択肢: 1問
- 型: 犯人特定（消去法 + 命令トレース）
- スプライト: yuuki→boy-b, mei→girl-b, takumi→boy-d, kotone→girl-c, sensei→teacher-f

### 気づき
- **命令トレースがVNの核心**: 命令カードを1枚ずつ「実行」する演出が、プログラムの順次実行そのもの。「カード①を実行→カード②を実行→左に曲がった！→花瓶にぶつかった！」の流れは、VNの「1クリック1ステップ」と完璧に噛み合う。他のシナリオにはない独自の体験。
- **正しい実行 vs バグ実行の対比**: evidence-3でバグ版のトレースを見せた後、「もし正しかったら」の正常ルートも提示。同じ命令カードで結果が変わることを体感させる演出が効果的。
- **全員に「落ち度」がある構造**: ユウキ（カード置きっぱなし）、タクミ（アーム不調隠し）、コトネ（気づいたのに言えず）、メイ（犯人）——全員がバグの連鎖に関与している。VNではこれを真実パートで1人ずつ明かすことで、単純な犯人当てを超えた深みが出る。
- **「テスト後の変更」の概念**: テスト（水曜日）で正常→本番（木曜日）でバグ。「テスト後に変更されたらバグが入る」というソフトウェア開発の原則を、物語の中で自然に学べる。先生のセリフで明示。

### 課題
- **背景の種類**: このシナリオは教室（classroom）と廊下（hallway）の2種類だけで足りる。学校系の汎用背景で対応可能。特殊背景は不要。
- **ろうかマップのビジュアル化**: 命令トレースのシーン（evidence-3）は、テキストだけでロボットの動きを追わせている。理想的にはろうかマップの背景画像（ロボットの動く方向が分かるもの）があると格段に良くなる。人間タスクとして専用の背景画像が欲しい。
- **選択肢が1問**: science-01やmath-01は2問（犯人+根拠）だったが、programming-01は犯人当て1問のみ。「なぜメイだと思うか」の根拠選択肢を追加してもいいかもしれない（次回検討）。
- **GM/先生キャラの設計**: overview.mdに先生キャラの名前がなかったため、汎用の「先生」（teacher-f）を使用。シリーズで先生キャラを統一するか、シナリオごとに変えるかは要検討。

## programming-02（バグ探偵団 Vol.2）

### 気づき
- **programmingシリーズはVol.ごとにキャラが異なる**: Vol.1はユウキ/メイ/タクミ/コトネ、Vol.2はアキラ/ナツ/カイ/リコ。スプライトは同じ役割タイプ→同じ素体のマッピングルールで統一。
- **パターン認識の推理がVN映え**: 「5, 10, 15, 20, ?」→+5→25という計算をnarrationで段階的に見せ、「答えカードは23——書きかえられた！」のflash+discoverが効果的。
- **ペンの色が物的証拠**: Card5で「赤いペン＝ナツ」が確定。他シリーズの「操作ログ」「筆跡」に相当する分かりやすい物証。
- **サプライズを守りたかった動機**: 悪意ゼロの善意犯。折り紙のプレゼントというかわいい動機がVNのエピローグで温かい着地を生む。

## programming-03（バグ探偵団 Vol.3）

**変換日**: 2026-03-24 / シーン16 / キャラ5 / 選択肢2問 / souma→boy-b, saya→girl-c, ren→boy-d, noa→girl-b, sensei→teacher-f

### 気づき
- **条件分岐トレースがVNの核心**: 「もし土がかわいていたら→水を出さない」を1ステップずつnarrationで追跡。正しいルール（出す）との比較でflash+discoverが効果的。
- **「出す」→「出さない」の2文字変更が衝撃**: シンプルだが条件分岐の本質を突く。1つの条件を変えるだけで結果が大きく変わることを実感させる。
- **またキャラが異なる**: Vol.1=ユウキ/メイ/タクミ/コトネ、Vol.2=アキラ/ナツ/カイ/リコ、Vol.3=ソウマ/サヤ/レン/ノア。スプライトの役割マッピングは統一。
- **花壇（schoolyard）背景**: プロローグとエピローグで校庭を使用。教室での推理パートとの対比が良い。

### 気づき
- **条件制御実験（変数の制御）がプログラミングと理科の横断**: 「変えるのは1つだけ」ルールが謎の核心。VNでは鉢A-Dの条件表を段階的に開示し、鉢Bの矛盾をflash+discoverで強調。

- **「予想が外れる＝失敗ではない」のメッセージ**: programmingシリーズとして初の「科学的方法」テーマ。エピローグの先生のセリフが教育的に深い。
